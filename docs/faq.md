# ImmiDock 常见问题

## 1. Docker 未安装

**现象：**

运行 `immidock doctor` 提示 Docker 不可用。

**解决：**

安装 Docker，并确保 `docker` 命令可用，然后再次运行：

```bash
immidock doctor
```

## 2. 恢复失败

**现象：**

`immidock restore` 报错或中断。

**排查：**

- 确认迁移包完整（可重新生成）
- 确认目标服务器空间充足
- 先运行 `immidock doctor` 检查环境

## 3. 端口冲突

**现象：**

容器启动失败，提示端口被占用。

**解决：**

- 检查目标服务器是否已有占用端口的服务
- 修改容器端口映射（或临时关闭占用进程）

## 4. 磁盘空间不足

**现象：**

迁移包创建失败或恢复失败。

**解决：**

- 清理无用镜像和容器
- 扩容磁盘
- 使用 `immidock doctor` 预先检查磁盘空间

## 5. 网络冲突

**现象：**

网络创建失败或容器无法加入网络。

**解决：**

ImmiDock 会自动处理常见子网冲突。若仍失败：

- 查看目标服务器已有网络：

```bash
docker network ls
```

- 删除无用网络后重试

---

建议遇到问题先运行：

```bash
immidock doctor
```

可以快速定位环境问题。
